name: semantic-pr
description: "Creates or edits a Pull Request from the current branch to a target branch. Analyzes commits, asks for context, and generates a well-structured PR with conventional commit prefix in the title. Use when the user asks to create a PR, open a PR, or edit an existing PR."
---

### Workflow

#### Step 1 - Identify branch, target, and template

1. Run `git branch --show-current`.
2. Check for a project PR template at (in priority order): `.github/pull_request_template.md`, `.github/PULL_REQUEST_TEMPLATE.md`, `pull_request_template.md`, `PULL_REQUEST_TEMPLATE.md`, `.github/PULL_REQUEST_TEMPLATE/*.md`.
3. If found, read it and ask: **"O projeto possui um template de PR (`<path>`). Deseja usar o template do projeto ou o template padrão da skill?"** If multiple exist in `PULL_REQUEST_TEMPLATE/`, list them and ask the user to choose. Store the choice for Step 4.
4. Ask for the **target branch** (if not provided), a **brief explanation** of the task/feature, and the **Jira task link** (skip the field entirely if not provided — no placeholder).

#### Step 2 - Check for existing PR

Run `gh pr list --head <current_branch> --base <target_branch> --state open --json number,title,url`. If a PR exists, **edit** it; otherwise **create** a new one.

#### Step 3 - Analyze commits

Run in parallel:
- `git log <target_branch>..<current_branch> --oneline`
- `git diff <target_branch>...<current_branch> --stat`

If no commits ahead of the target, inform the user and stop.

#### Step 4 - Build PR content

**Title format:** `type(scope): descrição breve`
- `type`: one of `feat|fix|refactor|docs|chore|style|perf|test|build|ci|revert`
- `scope`: optional but recommended
- PT-BR by default, imperative mood ("adicionar", "corrigir"), **max 69 characters**.

**Body — template selection:**

**A) Project template chosen:** load `references/project-template-rules.md` and follow it.

**B) Default template (or no project template):**

```markdown
## 📝 Descrição

<Parágrafo sintetizando a explicação do usuário e a análise dos commits.>

## 🔗 Relacionados

- **Task:** <link do Jira>

## 💡 Mudanças Realizadas

- <um item por commit, reescrito de forma clara em PT-BR>
```

Rules:
- **Descrição**: synthesize user explanation + commit analysis into a cohesive paragraph.
- **Relacionados**: omit the entire section (including header) if no Jira link was provided.
- **Mudanças Realizadas**: one item per commit, rewritten as human-readable PT-BR (not raw commit messages).

#### Step 5 - Confirm

Present the full PR (title + body) and wait for explicit confirmation. **Do NOT proceed otherwise.**

#### Step 6 - Create or edit

- Create: `gh pr create --base <target_branch> --title "<title>" --body "<body>"`
- Edit: `gh pr edit <pr_number> --title "<title>" --body "<body>"`

Show the PR URL to the user.

### Language

Default PT-BR; if the user explicitly asks for English, write title and body in English (template headers unchanged).

### Rules

- Title must have a conventional commit prefix and ≤ 69 chars (shorten description if needed while keeping meaning).
- Description must not be empty or generic.
- Never include AI attribution, co-authorship metadata, or "Generated by" footers.

For examples, read `references/examples.md` only if needed.
